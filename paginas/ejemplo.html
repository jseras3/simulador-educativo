🖼️ Arquitectura Cliente-Servidor

La arquitectura cliente-servidor es el modelo fundamental sobre el que funciona la Web. El cliente (tu navegador) solicita recursos y el servidor los provee. Esta separación permite que millones de usuarios accedan a la misma información simultáneamente.

📋 Resumen: Los 5 pasos del ciclo web

  1. El usuario escribe una URL y presiona Enter en su navegador.
  2. El navegador consulta el DNS para obtener la IP del servidor.
  3. Se establece la conexión HTTP/HTTPS con el servidor remoto.
  4. El servidor procesa la solicitud y devuelve el HTML, CSS y archivos.
  5. El navegador renderiza el código y muestra la página en pantalla.

📊 Protocolos web y sus características

Protocolo Puerto Seguridad Uso
HTTP/1.1 80 ❌ Sin cifrado Sitios legados, intranets
HTTPS 443 ✅ TLS/SSL Todos los sitios modernos
HTTP/2 443 ✅ TLS/SSL Multiplexación, compresión
HTTP/3 443 ✅ QUIC Conexiones ultra-rápidas

📝 Reflexión final

Comprender cómo funciona la Web no solo es útil para desarrolladores: también permite a cualquier usuario tomar mejores decisiones sobre su seguridad en línea, entender por qué algunas páginas cargan más rápido que otras, y apreciar la increíble infraestructura tecnológica que conecta al mundo.

Cada vez que visitas una página web, estás participando en un sofisticado diálogo entre tu dispositivo y computadoras repartidas por todo el planeta, en el que millones de paquetes de datos viajan a la velocidad de la luz para entregarte exactamente lo que buscas.
