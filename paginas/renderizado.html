🎨 ¿Qué es el renderizado?

El renderizado es el proceso mediante el cual el navegador convierte el código HTML, CSS y JavaScript que recibe del servidor en la página visual e interactiva que ves en tu pantalla.

Este proceso ocurre en varias etapas: primero el navegador parsea (analiza) el HTML para construir el DOM (Document Object Model), luego procesa el CSS para crear el CSSOM, los combina en un árbol de renderizado y finalmente pinta cada píxel en pantalla.

  • 📜 1. Parseo HTML
  • 🎨 2. CSS
  • 🌳 3. Árbol Render
  • 🖼️ 4. Pintar

📜 Parseo HTML

El motor del navegador lee el HTML línea a línea y construye el árbol DOM con todos los nodos del documento. Es la primera y más importante fase del renderizado.

🎨 Cálculo CSS

Se procesan las hojas de estilo y se calcula qué estilos aplican a cada elemento del DOM. El resultado es el CSSOM (CSS Object Model).

🌳 Árbol de Render

DOM + CSSOM se fusionan en el árbol de renderizado, que solo incluye los elementos visibles en pantalla (excluye display:none, etc.).

🖼️ Pintar píxeles (Paint)

El navegador calcula posición y tamaño de cada elemento (Layout) y dibuja cada píxel en pantalla (Paint). ¡Aquí nace la página visual!

🏗️ Etiquetas semánticas HTML5

HTML5 introdujo etiquetas semánticas: elementos con significado propio que describen el rol del contenido. Esto mejora la accesibilidad, el SEO y la legibilidad del código.

Etiqueta Función
<header> Encabezado de la página o sección. Suele contener el logo y la navegación.
<nav> Bloque de enlaces de navegación. Ayuda a lectores de pantalla a identificar el menú.
<main> Contenido principal único de la página. Solo debe haber uno por documento.
<section> Agrupa contenido temáticamente relacionado dentro de la página.
<article> Contenido independiente y reutilizable, como una noticia o entrada de blog.
<aside> Contenido secundario relacionado, como una barra lateral o notas adicionales.
<footer> Pie de página con información de contacto, derechos de autor y enlaces legales.

✅ ¿Por qué usar etiquetas semánticas?

Las etiquetas semánticas comunican significado tanto al navegador como a motores de búsqueda y tecnologías de asistencia.

Un lector de pantalla para personas con discapacidad visual puede identificar automáticamente el menú de navegación (<nav>) o el contenido principal (<main>).
